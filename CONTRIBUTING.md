## Guidelines
1. Ensure that your code is formatted correctly (4 spaces for indentation) and follows the general style of code in this project
2. Ensure that there are no offensive or NSFW material
3. Use CommonJS files for the server (.cjs)
4. Follow the [Material 3](https://m3.material.io) themeing for the website and use [Material Web](https://m3.material.io/develop/web) components for buttons, dividers, icons, inputs etc.

## Checklist
Before you submit a pull request, please ensure the following:
* [ ] I have ensured that there is a useful contribution involved (e.g. fixing bugs, adding new features) and not miniscule changes (e.g. editing a single word/letter in the README)
* [ ] I have ensured that the code folllows the general formating this project
* [ ] I have ensured that there are no NSFW materials
* [ ] I have ensured that my pull request follows all the guidelines stated

## Submit a Pull Request
1. Fork this repository
2. Make your changes
3. Click ```Compare & pull request```
4. Complete checklist in the template
5. Wait for the pull request to be merged or rejected
